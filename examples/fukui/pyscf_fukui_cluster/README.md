# PySCF Fukui Function Calculations — Cluster Package

Self-contained folder for computing electron density and Fukui functions
(f+, f-, f0) for isolated molecules using PySCF with Gaussian basis sets.

Uses **def2-SVP** (double-zeta) basis set and **PBE** functional by default.

## Contents

```
pyscf_fukui_cluster/
├── generate_jobs.py     # Job generator: bakes standalone scripts from geometries
├── README.md            # This file
├── geometries/          # XYZ files for all molecules (planar, z=0)
│   ├── H2O.xyz
│   ├── CH2O.xyz
│   ├── CH2NH.xyz
│   ├── C2H4.xyz
│   ├── pyrrol.xyz
│   ├── pyridine.xyz
│   ├── pentacene.xyz
│   └── PTCDA.xyz
└── jobs/                # Baked scripts (generated by generate_jobs.py)
    ├── geometries/      # Copy of geometries for cluster
    ├── run_<mol>_N.py   # 24 single-point scripts (8 molecules × 3 charge states)
    ├── run_<mol>_A.py   #   each runs ONE PySCF calculation, saves rho_<TAG>.cube + .npy
    ├── run_<mol>_C.py
    ├── submit_<mol>_N.pbs  # 24 PBS scripts with per-molecule resources
    ├── submit_<mol>_A.pbs
    ├── submit_<mol>_C.pbs
    ├── postprocess_<mol>.py  # 8 post-process scripts: reads 3 .npy, computes Fukui
    ├── submit_<mol>_post.pbs # 8 PBS scripts for post-processing (1 CPU, 15 min)
    └── results/            # Output (created after jobs run)
        └── <mol>_<xc>_<basis>/
            ├── rho_N.cube / .npy     # Neutral density
            ├── rho_A.cube / .npy     # Anion (N+1) density
            ├── rho_C.cube / .npy     # Cation (N-1) density
            ├── N.txt / A.txt / C.txt # PySCF SCF logs (stdout)
            ├── fukui_f_plus.npy      # f+ = ρ(N+1) - ρ(N)  [post-processing]
            ├── fukui_f_minus.npy     # f- = ρ(N) - ρ(N-1)  [post-processing]
            └── fukui_f_zero.npy      # f0 = 0.5(f+ + f-)   [post-processing]
```

## What It Computes

For each molecule, three single-point DFT calculations at the **same fixed geometry**:

| State | Charge | Spin (PySCF) | Description |
|-------|--------|-------------|-------------|
| N     | 0      | 0 or 1*     | Neutral molecule |
| N+1   | -1     | 1 or 0*     | Anion (added electron) |
| N-1   | +1     | 1 or 0*     | Cation (removed electron) |

\* PySCF spin = nalpha - nbeta. For even-electron molecules: neutral spin=0
(singlet), anion/cation spin=1 (doublet). For odd-electron molecules: reversed.

Each single-point job saves the electron density as `.cube` (via `cubegen.density`)
and `.npy` (read back from cube). Fukui functions are computed in post-processing:

- **f+** (electrophilic) = ρ(N+1) - ρ(N)
- **f-** (nucleophilic) = ρ(N) - ρ(N-1)
- **f0** (radical)      = 0.5 × (f+ + f-)

## How to Run

### Generate jobs (already done, but to regenerate)

```bash
python generate_jobs.py                              # default: def2-SVP, PBE
python generate_jobs.py --basis def2-TZVP --xc PBE   # triple-zeta
python generate_jobs.py --basis def2-SVP --xc B3LYP  # different functional
```

### Prerequisites on Metacentrum

PySCF is NOT available as a module. Install via pip:

```bash
module add mambaforge
pip install --user pyscf
```

### Submit on Metacentrum

```bash
cd jobs/

# Submit all 24 single-point jobs (independent, run in parallel)
for f in submit_*_N.pbs submit_*_A.pbs submit_*_C.pbs; do qsub $f; done

# After all 3 densities for a molecule are done, submit post-processing
qsub submit_H2O_post.pbs
# ... or all at once:
for f in submit_*_post.pbs; do qsub $f; done
```

### Submit a single job

```bash
qsub submit_H2O_N.pbs
```

### Run locally (if PySCF installed)

```bash
python3 jobs/run_H2O_N.py
python3 jobs/run_H2O_A.py
python3 jobs/run_H2O_C.py
python3 jobs/postprocess_H2O.py
```

## Resource Estimates

PySCF with Gaussian basis sets is all-electron, no periodic box needed.
Memory scales with number of basis functions (~N² for density matrix).

| Molecule  | Atoms | Electrons | Basis funcs (~def2-SVP) | CPUs | Memory  | Walltime |
|-----------|-------|-----------|------------------------|------|---------|----------|
| H2O       | 3     | 10        | ~25                    | 4    | 8 GB    | 1 h      |
| CH2O      | 4     | 12        | ~40                    | 4    | 8 GB    | 1 h      |
| CH2NH     | 5     | 14        | ~45                    | 4    | 8 GB    | 1 h      |
| C2H4      | 6     | 12        | ~48                    | 4    | 8 GB    | 1 h      |
| pyrrol    | 10    | 26        | ~100                   | 4    | 8 GB    | 2 h      |
| pyridine  | 11    | 26        | ~110                   | 4    | 8 GB    | 2 h      |
| pentacene | 36    | 156       | ~360                   | 8    | 16 GB   | 8 h      |
| PTCDA     | 38    | 188       | ~420                   | 8    | 16 GB   | 8 h      |

### Memory scaling

PySCF memory depends on:
- **Density matrix**: N_basis² × 8 bytes
- **Two-electron integrals**: can be large for big molecules, but PySCF uses
  density fitting / out-of-core algorithms automatically when needed
- **Cube grid**: (grid_points) × 8 bytes — typically 100³ = 8 MB

**Rule of thumb:** ~2 GB per 100 basis functions. For pentacene/PTCDA
(~400 basis functions), 16 GB is safe.

### CPU scaling

PySCF parallelizes via OpenMP (threaded BLAS). Set `OMP_NUM_THREADS` to
match allocated CPUs. No MPI needed.
- Small molecules (≤6 atoms): 4 cores
- Medium (10-11 atoms): 4 cores
- Large (36-38 atoms): 8 cores

## Key Parameters

| Parameter     | Default   | Description |
|---------------|-----------|-------------|
| `--basis`     | def2-SVP  | Double-zeta basis set. def2-TZVP for triple-zeta |
| `--xc`        | PBE       | XC functional |
| `--resolution`| 0.15 Bohr | Cube grid spacing (smaller = finer grid) |
| `--margin`    | 4.0 Bohr  | Extra space around molecule in cube |

### Basis set notes

- **def2-SVP**: double-zeta, ~2-3 basis functions per atom. Fast, good for testing.
- **def2-TZVP**: triple-zeta, ~5-6 per atom. Better quality, 3-5x slower.
- **def2-SVPD / def2-TZVPD**: with diffuse functions. Important for anions
  (diffuse electron density). Consider for production anion calculations.

### Cube grid

- `resolution=0.15` Bohr (~0.08 Å) is fine enough for visualization.
- `resolution=0.10` for higher quality (2-3x larger files).
- `margin=4.0` Bohr (~2.1 Å) around molecule. Increase to 6.0 for diffuse
  anion density.

## Sanity Checks

After post-processing, check the printed sums:
- **sum f+ / sum f- ratio should be ~1.0** (one electron added/removed)
- If ratio deviates by >5%, check cube grid resolution and margin

## Differences from GPAW package

| Aspect | PySCF | GPAW |
|--------|-------|------|
| Basis | Gaussian (def2-SVP) | Plane waves (PW 500 eV) |
| Periodicity | Isolated molecule | Periodic box + vacuum |
| All-electron | Yes | Yes (PAW reconstruction) |
| Parallelism | OpenMP (threads) | MPI |
| Module on Metacentrum | pip install | `module add gpaw` |
| Memory | ~N_basis² | ~ecut^(3/2) × cell_volume |
| Best for | Small-medium molecules | Large systems, surfaces |

## Notes

- All molecules are planar and aligned in the xy-plane (z=0)
- Each baked `run_*.py` script is fully self-contained — hardcoded
  geometry, charge, spin. No CLI args, no external deps beyond PySCF + NumPy
- Post-processing (`postprocess_*.py`) only needs NumPy
- UKS density matrices are summed (alpha+beta) before cube generation
  (fixes known PySCF `cubegen.density()` issue with UKS)
- `OMP_NUM_THREADS=$PBS_NUM_PPN` is set in PBS scripts for OpenMP parallelism

## CO Rigid Scan Jobs

PySCF CO rigid scan jobs are generated by `generate_CO_scan_jobs.py` in the
sibling `gpaw_fukui_cluster/` directory (with `--pyscf` flag). The baked
scripts are written to `gpaw_fukui_cluster/jobs_CO_scan_pyscf/` and use
PySCF DFT with `density_fit()` for efficiency. See the
[gpaw_fukui_cluster README](../gpaw_fukui_cluster/README.md#2-co-rigid-scan)
for details.
