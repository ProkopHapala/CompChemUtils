# GPAW Fukui Function Calculations — Cluster Package

Self-contained folder for computing electron density and Fukui functions
(f+, f-, f0) for isolated molecules using GPAW (plane-wave PBE).

## Two Ways to Run

### 1. Baked jobs (recommended for cluster)

Pre-generated standalone scripts — one per molecule per charge state.
Each script has hardcoded geometry and parameters, no CLI args, no loops.
Generated by `generate_jobs.py`.

### 2. Flexible runner (for local testing or custom params)

`run_fukui.py` takes CLI args (`--mol`, `--ecut`, `--vacuum`, `--batch`).
`submit.pbs` + `submit_all.sh` submit via PBS with per-molecule resources.

## Two Workflows

### 1. Fukui Function Calculations

Pre-generated standalone scripts — one per molecule per charge state.
Each script has hardcoded geometry and parameters, no CLI args, no loops.
Generated by `generate_jobs.py`.

### 2. CO Rigid Scan

Scans CO molecule perpendicular to molecular plane over symmetry-inequivalent
heavy atoms. Computes interaction energy E_int(r) = E_total - E_mol - E_CO.
Generated by `generate_CO_scan_jobs.py`.

Supports both GPAW (plane-wave, periodic box) and PySCF (Gaussian basis,
isolated molecule) backends. Use `--pyscf` flag to also generate PySCF jobs.

```bash
# Generate GPAW scan jobs only
python generate_CO_scan_jobs.py --ecut 500 --vacuum 12

# Generate both GPAW and PySCF scan jobs
python generate_CO_scan_jobs.py --pyscf --basis def2-SVP

# Flexible runner (single molecule, local testing)
python run_CO_scan.py --mol C2H4 --ecut 200 --vacuum 8.0
```

Scan scripts save results incrementally to `_scan.dat` (flushed after each
point) to prevent data loss on crashes.

## What It Computes

For each molecule, three single-point DFT calculations are performed
at the **same fixed geometry**:

| State | Charge | Spinpol | Description |
|-------|--------|---------|-------------|
| N     | 0      | False*  | Neutral molecule |
| N+1   | -1     | True*   | Anion (added electron) |
| N-1   | +1     | True*   | Cation (removed electron) |

\* For even-electron molecules (all in our list): neutral is singlet
(spin-unpolarized), anion/cation are doublets (spin-polarized).

Each single-point job saves only the raw all-electron density as `.npy`.
Fukui functions are computed in a separate post-processing step:

- **f+** (electrophilic) = ρ(N+1) - ρ(N) — where an added electron goes
- **f-** (nucleophilic) = ρ(N) - ρ(N-1) — where electron density is lost
- **f0** (radical)      = 0.5 × (f+ + f-)

## How to Run — Baked Jobs (cluster)

### Generate jobs (already done, but to regenerate with different params)

```bash
python generate_jobs.py --ecut 500 --vacuum 12    # production
python generate_jobs.py --ecut 200 --vacuum 8     # quick test
```

### Submit on Metacentrum

```bash
cd jobs/

# Submit all 24 single-point jobs (independent, run in parallel)
for f in submit_*_N.pbs submit_*_A.pbs submit_*_C.pbs; do qsub $f; done

# After all 3 densities for a molecule are done, submit post-processing
qsub submit_H2O_post.pbs
qsub submit_pentacene_post.pbs
# ... or all at once:
for f in submit_*_post.pbs; do qsub $f; done
```

### Submit a single job

```bash
qsub submit_H2O_N.pbs
```

### Check status

```bash
qstat -u prokop
cat fukui_H2O_N.oJOBID    # stdout
```

## How to Run — Flexible Runner (local or custom)

### Prerequisites on Metacentrum

```bash
module add gpaw
```

GPAW comes with ASE and NumPy bundled. No pip installs needed.

### Submit a single molecule

```bash
# Small molecule (default resources: 4 CPUs, 8 GB, 2h)
qsub -v mol=H2O submit.pbs

# Large molecule (override resources)
qsub -v mol=pentacene -l select=1:ncpus=16:mem=32gb:scratch_local=20gb -l walltime=12:00:00 submit.pbs
```

### Submit all molecules at once

```bash
chmod +x submit_all.sh
./submit_all.sh
```

### Run locally (if GPAW installed)

```bash
export OMP_NUM_THREADS=1
python3 run_fukui.py --mol H2O --ecut 500 --vacuum 12.0
python3 run_fukui.py --batch --ecut 500 --vacuum 12.0
```

### Run interactively on Metacentrum (for testing)

```bash
qsub -I -l walltime=00:30:00 -l select=1:ncpus=4:mem=8gb
module add gpaw
export OMP_NUM_THREADS=1
python3 run_fukui.py --mol H2O --ecut 200 --vacuum 8.0
```

### How memory scales

GPAW plane-wave memory depends on:
- **Number of plane waves** ~ (ecut)^(3/2) × cell_volume
- **Number of bands** ~ number of electrons / 2
- **Grid size** for real-space density ~ cell_volume / grid_spacing³

At 500 eV with 12 Å vacuum, the grid is ~80×80×50 = 320k points.
Each double array = 2.5 MB. Total density arrays ~ 20 MB per state.
The plane-wave coefficient arrays dominate for large molecules.

**Rule of thumb:** ~4 GB per 100 atoms at 500 eV. For pentacene/PTCDA
(~38 atoms), 32 GB is safe. For small molecules, 8 GB is plenty.

### CPU scaling

GPAW parallelizes well with MPI up to ~16 cores. Beyond that,
diminishing returns for isolated molecules (gamma-point only).
- Small molecules (≤6 atoms): 4 cores sufficient
- Medium (10-11 atoms): 8 cores
- Large (36-38 atoms): 16 cores

## Key Parameters

| Parameter  | Default | Description |
|------------|---------|-------------|
| `ecut`     | 500 eV  | Plane-wave cutoff. 200 for quick tests, 500 for production |
| `vacuum`   | 12 Å   | Vacuum padding around molecule. 8 for tests, 12 for production |
| `xc`       | PBE     | Exchange-correlation functional |

### Cutoff convergence

For reference-quality densities, 500 eV is recommended. Quick tests
can use 200 eV (3-5x faster, ~5% error in density). Convergence can
be checked by comparing integrated Fukui values at different cutoffs.

### Vacuum convergence

For isolated molecules, sufficient vacuum prevents image-image
interaction through periodic boundaries. 12 Å is standard. For
charged species (anion/cation), more vacuum may be needed if the
integrated Fukui values deviate significantly from 1.0.

## Sanity Checks

After post-processing, check the printed sums:
- **sum f+ / sum f- ratio should be ~1.0** (one electron added/removed)
- If ratio deviates by >5%, increase vacuum or ecut

## Notes

- All molecules are planar and aligned in the xy-plane (z=0)
- Each baked `run_*.py` script is fully self-contained — hardcoded
  geometry, cell, charge, spinpol. No CLI args, no external deps beyond
  GPAW + ASE + NumPy
- Post-processing (`postprocess_*.py`) only needs NumPy — can run on
  a 1-CPU node in 15 minutes
- Spin: closed-shell molecules use spin-unpolarized neutral,
  spin-polarized anion/cation (doublet states)
- Fermi-Dirac smearing of 0.05 eV helps SCF convergence for
  near-degenerate frontier orbitals
- `OMP_NUM_THREADS=1` is set in PBS scripts to avoid conflicts
  with MPI parallelization
- Density is saved in Å⁻³ (GPAW native units). Convert to Bohr⁻³
  with `rho * B2A³` if needed for cube file post-processing
